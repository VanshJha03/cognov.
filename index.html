Overview

CognoV is a backend accounting engine. Your inventory app calls our API on every business event. We compute double-entry journal entries and persist them. You query them back whenever needed.

Base URL
https://cognov-3a24.vercel.app/request
Authentication

Pass your 10-character Developer API Key as api_key in the JSON body or as a query parameter. All data is scoped to your key.

How You Receive the Output

CognoV is synchronous. Every POST request you make returns accounting output immediately in the HTTP response body. Your inventory app does not poll or wait — it reads the response JSON and the accounting is already done.

Sales event response

Your inventory app records a sale. It POSTs to /transaction. The HTTP response body contains: 

{
  "debit": "Cash",
  "credit": "Sales Revenue",
  "revenue": 950.00,
  "cogs": 500.00
}

debit: account debited · credit: account credited · revenue: total revenue posted · cogs: cost of goods sold deducted from inventory

Purchase event response 
{
  "debit": "Inventory",
  "credit": "Cash",
  "amount": 250.00
}
Credit sale response (sales_on_credit) 
{
  "debit": "Accounts Receivable",
  "credit": "Sales Revenue",
  "revenue": 1200.00,
  "cogs": 600.00
}
Fetching stored reports (Journal, Balance Sheet, Trial Balance)

required

POST /register_user

POST /register_user?api_key={your_key}

Create a new User ID for a business in your app. Each business gets its own isolated accounting ledger.

Response

{
  "user_id": "a1b2c3d4"
}

Store this user_id in your database. Pass it with every transaction for this business.

POST /transaction

POST /transaction

The core endpoint. Send any business event. Accounting is computed and returned immediately in the response.

Request Body

{
  "api_key": "X9z2LpQ4R1",
  "user_id": "a1b2c3d4",
  "transaction": {
    "action": "...",
    ...fields depend on action...
  }
}

Action Types

initialize
Set opening inventory, assets, liabilities. Do this once per user before any transactions.
purchase
item, qty, unit_cost — cash purchase, debits Inventory.
purchase_on_credit
item, qty, unit_cost, creditor — credits Accounts Payable.
sales
item, qty, selling_price — cash sale, debits Cash.
sales_on_credit
item, qty, selling_price, debtor — debits Accounts Receivable.
purchase_return
item, qty, unit_cost — returns inventory, credits Cash.
sales_return
item, qty, selling_price — reverses a sale.
write_off
account_to_debit, account_to_credit, amount, description
adjust
debit_account, credit_account, amount, description

GET /fetch/{user_id}/{report}

GET /fetch/{user_id}/{report}?api_key={your_key}

Retrieve full accounting statements for any user at any time.

journal
report
Full list of every debit/credit posting with timestamps and amounts.
trialbalance
report
Running total of debits and credits for all accounts.
balancesheet
report
Assets, Liabilities, and Equity as a structured JSON object.

Error Codes

200 OK
Transaction processed. Ledger written. Response contains journal output.
400
Bad request — missing field, insufficient inventory, or unknown action type.
401
Invalid or missing API key.
403
API key does not own this user_id.
409
Entity already registered. Use your existing API key.
429
Rate limit exceeded — 100 requests/minute per user_id.

Privacy Policy

Effective June 14, 2026

1. What we store

We store API keys, User IDs, transaction events (action type, amounts, timestamps), journal entries, trial balance figures, and balance sheet snapshots. We do not store any end-user personal data, names, or payment credentials.

2. Data isolation

Every Developer API Key is completely isolated. Data belonging to one key is never accessible to another. Each User ID under a key maintains its own accounting ledger.

3. Security

All data is stored encrypted at rest via Supabase (AES-256). All transport uses TLS 1.3. API keys are generated using cryptographically secure random functions.

4. Retention

Transaction records are retained for 7 years for accounting compliance. You may request full deletion at any time by contacting security@cognov.com.

5. Third parties

We use Supabase for database hosting and Cloudflare for compute. No data is sold, rented, or shared with any other third parties.
